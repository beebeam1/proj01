# proj01
김성민의 과제 프로젝트 01. 구르미로 랜딩페이지.


1. 프로젝트 개요.

기존에 보유하던 도메인을 파킹페이지로 임시 연결해놨었는데 개인용 랜딩페이지를 새롭게 작성해봤습니다.
웹호스팅을 별도로 사용하려다가 간단한 페이지라 공유기에 물려놓은 외장하드를 웹서버로 활용. 외부에서 도메인주소로 바로 열리게끔 설정을 했습니다.


2. 프로젝트 목적.

개인용 sns들을 모아놓은 중계용 허브페이지로 초기에는 파킹페이지에 url 을 나열하는 정도로도 충분했지만 사업용으로 성격이 변화되면서 개선이 필요했습니다.
헤더와 푸터를 추가할까 고민했는데 완전한 비즈니스 성격을 부여하는것 보다는 일단 기능에 충실하고 개인용에 가까운 가볍고 좋은 인상을 주도록 만들자는 심플한 목적을 정했습니다.
 

3. 프로젝트 동기.

이번 교육때 많은것들을 배우다보니 이것을 가장 먼저 손봐야겠다는 생각이 들었습니다.
1만시간 프로젝트 페이지와 구조와 성격이 유사해 기왕 하는김에 이것을 손대야겠다는 생각이 들었네요.


4. 개발 과정.

4-1. 기술적 측면.
십수년전 직접 홈페이지를 만들어본적이 있고 이번 교육에서 최근의 시멘틱 개발트랜드와 html, css 를 배우긴 했지만 아직은 익숙하지 않다보니 직접 코드를 짜는것 보다는 제미나이에게 물어보고 만들어준 코드에서 유익한 부분을 추가해 짜집기하는 방식으로 만들었습니다.
제미나이가 요청을 엉뚱하게 받아들이거나 해석해 이상한 결과물을 내놓는 경우가 많았지만 이번 교육을 통해 배운 코드의 구조와 문법에 대한 이해가 큰 도움이 되었습니다.
애초 목적이 가능한 심플한 페이지였기에 굳이 css 파일과 폴더를 구분해 구성하지는 않았습니다.
하지만 차후 개선을 하게 될 경우를 대비해 제미나이가 만들어준 주석문들은 그대로 넣었습니다.

4-2. 디자인적 측면.
애초 목적이 여러 sns로 흩어진 개인용 페이지들의 url을 찾아가기 쉽게 모은 페이지라 처음에는 파킹페이지에서 그랬던것처럼 똑같은 문구와 똑같은 url들을 넣고 배경에 이미지만 하나 깔아놓는 방식으로 구상했습니다.
그런데 제미나이가 여러 최근 트랜드를 고려해 좀더 깔끔한 버튼식으로. 그리고 배경 이미지가 살짝 비치는 효과등을 넣어보라고 제안해주더군요.  코드 몇개만 수정하면 가능한 것들이라 사용을 했습니다.
결과적으로 매우 마음에 드는 형태가 되었는데 배경이미지를 제외한 디자인은 거의 전부 제미나이의 제안에 의한 것들입니다.
배경 이미지는 기존에 갖고있던 흑백 이미지를 이용해 챗gpt 에게 요청해 만들었는데 해상도와 사이즈 요청을 잘 받아들이지 못해서 코파일럿으로 구체적인 이미지 수정을 다시 해줬습니다.

4-3. 웹호스팅 문제.
처음에는 카페24 의 500원짜리 웹호스팅을 사용해볼까 했는데 용량작은 파일 하나와 이미지 하나가 전부라 항상 켜져있는 공유기 외장하드를 이용해보자는 생각이 들었습니다.
포워딩을 이용해 연결했는데 외장하드의 주소가 그대로 나오고 또한 http 비암호화 경고문구가 나와서 클라우드플레어의 ssl 서비스를 이용해봤는데 여기에서는 네임서버와 공유기 외장하드의 url 을 포워딩이 아닌 네임서버와 직접 연결해야 하고 그 타켓주소에는 숫자나 슬래시를 넣으면 안된다는 제약이 있더군요.
때문에 다시 클라우드플레어의 네임서버를 해제하고 원래 도메인 업체의 포워딩으로 연결했는데 이번에는 공유기가 물려있는 통신사에서 해당 포트를 막은것인지 한동안 잘 열리던 페이지가 갑자기 열리지않는 문제를 새롭게 발견했습니다.
잘 열리다가 갑자기 열리지 않는 문제가 발생해 현재 해결과정 중입니다.


5. 미흡한점. 발전계획

목표가 높지 않아서 작업은 어렵지 않았습니다. 장기적으로는 각각의 sns 에 흩어져있는 자료와 기능들을 하나로 합쳐나가려고 합니다.
기술적인 문제보다 디자인,컨셉,방향성이 가장 큰 과제인데 이쪽으로도 제미나이같은 ai서비스들이 큰 도움을 준다는것을 알게되어 앞으로도 적극적으로 사용해나가려고 합니다.
